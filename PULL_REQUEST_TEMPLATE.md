## PR 內容描述

本次 PR 主要包含以下 CI/CD 和監控的硬化更新：

1.  **CI/CD 增強**:
    *   新增 `db-smoke.yml` 工作流程，在每次 PR 和 push 到 `main`/`staging` 分支時，執行以下檢查：
        *   **Host 白名單與 Worker 路由檢查**: 請求 `https://api.morningai.me/health`，確保返回 200，否則標記為 "CF 路由風險" 並失敗。
        *   **資料庫連接測試**: 請求 `https://api.morningai.me/healthz`，確保資料庫連接正常。
        *   **禁止的依賴檢查**: 檢查專案中是否包含 `aiohttp` 依賴，確保代碼庫的純淨性。
        *   **API 回應時間檢查**: 監控 `/health` 端點的回應時間，並在超過 5 秒時發出警告。
        *   **安全標頭檢查**: 驗證 Cloudflare 和 CORS 相關的安全標頭。
    *   在現有的 `ci.yml` 中也加入了對 `aiohttp` 的檢查，確保所有 CI 流程都能覆蓋此規則。

2.  **監控配置指南**:
    *   創建了 `MONITORING_SETUP_GUIDE.md`，詳細說明如何配置：
        *   **Sentry** 用於錯誤追蹤。
        *   **UptimeRobot** 用於服務可用性監控。
        *   **Grafana** 用於性能指標可視化。

3.  **Runbook 更新**:
    *   創建了 `RUNBOOK_UPDATE.md`，補充了關於 Cloudflare Worker 變更的影響分析和詳細的回滾步驟，增強了系統的可維護性和應急響應能力。
